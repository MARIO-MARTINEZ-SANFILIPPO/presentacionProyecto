Principios de Privacidad desde el Diseño (PbD)

Aplicación en el Proyecto "Asistente de Compras por Voz"

1. Proactivo, no Reactivo

En lugar de esperar a un incidente de privacidad, hemos realizado este análisis desde la concepción del proyecto, utilizando herramientas como el Checklist "Responsibility by Design" para guiar nuestras decisiones.

2. Privacidad por Defecto

Esta es la piedra angular de nuestra elección de STT. La API nativa de Android no nos permite controlar el uso de los datos. Por ello, elegimos la API de Google Cloud Speech-to-Text, que nos permite desactivar explícitamente el registro de datos. Por defecto, la privacidad del usuario es máxima.

3. Privacidad Integrada

La seguridad no es un añadido, es parte de la arquitectura. Esto se refleja en nuestro stack con: HTTPS, JWT, R8, Wazuh/Falco para monitoreo y roles IAM en la base de datos.

4. Funcionalidad Total

Demostramos que la seguridad y la privacidad no son un obstáculo, sino un habilitador. Un usuario solo confiará datos sensibles a un sistema que percibe como seguro. La confianza es una funcionalidad clave de nuestro producto.

5. Seguridad Extremo-a-Extremo

Los datos están protegidos durante todo su ciclo de vida: desde la captura en el dispositivo, pasando por el transporte cifrado (HTTPS), hasta su almacenamiento seguro y con acceso restringido en Google Cloud SQL.

6. Visibilidad y Transparencia

Nos comprometemos a tener una política de privacidad clara y accesible. El consentimiento informado para el procesamiento de voz será explícito y audible durante el onboarding, explicando de forma sencilla qué se procesa y por qué.

7. Respeto por la Privacidad del Usuario

El sistema se diseñará para facilitar el ejercicio de los derechos ARCO (Acceso, Rectificación, Cancelación, Oposición). El panel administrativo incluirá funcionalidades para gestionarlos.
